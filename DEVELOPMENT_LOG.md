# 🛡️ 딸꾹뚝 (Hiccup Shield) 프로젝트 개발 요약 보고서

본 보고서는 '상황별 최적의 딸꾹질 완화 알고리즘'을 제공하는 인터랙티브 케어 가이드 앱, **딸꾹뚝**의 주요 개발 및 개선 사항을 정리한 문서입니다.

---

## 1. 프로젝트 개요
- **앱 이름**: 딸꾹뚝 (Hiccup Stop)
- **컨셉**: "The Gentle Shield (부드러운 방어막)" - 당황스러운 순간에 안도감을 주는 세련된 위트와 미니멀리즘 기반의 헬스케어 앱.
- **주요 타겟**: 갑작스러운 딸꾹질로 당황한 전 세계 사용자 (한/영 병기 지원).

## 2. 핵심 기능 및 구현 (3-Core Methods)

### 2-1. 숨참기 (Vacuum Hold)
- **과학적 원리**: $CO_2$ 농도를 높여 횡격막 경련 중단.
- **인터랙션**: 5-15-5 브리딩 타이머와 서클 애니메이션을 통한 시각적 가이드 제공.
- **특징**: iOS 사용자를 고려하여 타이머 매 초마다 시각적 펄스 피드백 구현.

### 2-2. 90도 음수 (Silent Sip) - *주요 개선 사항*
- **과학적 원리**: 역방향 자세로 미주신경 자극.
- **로직 최적화**: 
    - 기기 기울기 센서를 활용하여 75도 이상 숙였을 때만 "물을 마시는 중..." 상태 활성화.
    - 단순히 진입 시 질문하는 것이 아니라, **실제 동작(숙였다가 다시 일어남)을 완료한 후**에만 성공 여부를 묻도록 UX 개선.

### 2-3. 동서남북 (Master Ritual) - *주요 개선 사항*
- **과학적 원리**: 다방향 회전 및 음성 인식을 통한 신경계 리셋.
- **인터랙션**: 가상 나침반 UI와 Web Speech API 결합.
- **UI 개선**: 방향 지시 원(Indicator)이 동(0°), 서(180°), 남(270°), 북(90°)의 논리적 흐름에 맞춰 직관적으로 이동하도록 수정.
- **보안 강화**: 마이크 권한 상태 및 브라우저 지원 여부를 실시간으로 감지하여 안내 문구 출력.

## 3. UI/UX 디자인 시스템

### 3-1. 모바일 최적화 (Mobile-First Layout)
- **프레임 고정**: 모든 서브 페이지(마이 리포트, 뱃지 아카이브, 환경설정 등)가 웹 화면 전체로 퍼지지 않고 모바일 규격(`max-w-md`) 안에서만 보이도록 절대 좌표(`absolute`) 레이아웃 적용.
- **글래스모피즘**: Deep Navy 배경 위에 투명한 유리 느낌의 패널과 파스텔 톤(Sky Blue, Mint Green, Vibrant Orange) 포인트 컬러 사용.

### 3-2. 성능 및 속도 개선
- **스플래시 최적화**: 앱 초기 실행 시 로고 노출 시간을 **2.5초에서 1.2초로 단축**하여 사용자 체감 속도 향상.
- **애니메이션 가속**: 단축된 시간에 맞춰 로고 및 로딩 바의 애니메이션 속도를 Snappy하게 조정.

## 4. 기술적 사양 및 인프라

### 4-1. 기술 스택
- **Framework**: Next.js (App Router) / TypeScript
- **Styling**: Tailwind CSS v4 / Vanilla CSS 애니메이션
- **Web API**: DeviceOrientation (기울기), Web Speech (음성 인식), Vibration (햅틱)
- **PWA**: 오프라인 사용 및 홈 화면 설치 지원 (`manifest.json`, `sw.js`)

### 4-2. 테스트 및 배포
- **보안 컨텍스트**: HTTPS 환경 필수 (센서 및 마이크 작동 조건).
- **로컬 터널링**: 모바일 실기기 테스트를 위해 `localtunnel`을 통한 보안 터널 구축 지원.

## 5. 향후 확장 가능성
- **개인화 분석**: 누적된 성공 데이터를 바탕으로 사용자별 '가장 잘 듣는 방법' 추천 고도화.
- **뱃지 시스템**: 다양한 성취 업적(심야의 수호자, 방향의 지배자 등)을 통한 게이미피케이션 요소 강화.
