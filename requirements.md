# [기능 정의서] 프로젝트명: 딸꾹 쉴드 (Hiccup Shield)

## 0. 온보딩: 개인화 설정 (Onboarding: Profile Setup)
### 0-1. [Page] 이름 입력 화면
*   **기능:** 사용자가 앱에서 불리고 싶은 닉네임 또는 이름을 입력받는 최초 진입 화면.
*   **UI 문구 (KR/EN 병기):**
    *   **Question:** "반가워요! 당신을 어떻게 불러드릴까요?" / "Welcome! What should I call you?"
    *   **Input Placeholder:** "이름을 입력해 주세요" / "Enter your name"
*   **시스템 로직:**
    *   입력된 값을 `{User_Name}`이라는 변수로 저장.
    *   이후 모든 문구에서 `{User_Name}`을 호출하여 출력.

## 1. 제품 개요
*   **핵심 컨셉:** 상황별 최적의 딸꾹질 완화 알고리즘을 제공하는 ‘인터랙티브 케어 가이드’
*   **주요 목표:** 사용자가 처한 환경(물 유무, 장소)에 맞춰 고민 없이 즉각적인 솔루션 실행
*   **글로벌 최적화:** 전 세계 누구나 당황한 상황에서 즉각 이해할 수 있도록 직관적인 UI와 영문 병기(Global Bilingual UI) 제공

## 1.5 브랜드 아이덴티티 및 스플래시 (Brand Identity & Splash)
기존의 딱딱하고 차가운 의료용 앱 느낌을 탈피하고, 사용자가 당황한 순간 안도감과 친근함을 느낄 수 있도록 '세련된 위트'와 '미니멀리즘'을 결합합니다.

*   **최종 로고 컨셉: "The Gentle Shield (부드러운 방어막)"**
    *   **비주얼 구성:** 심리적으로 안정감을 주는 부드러운 화이트(Off-White) 라운드 스퀘어 배경. 입의 윤곽을 기하학적으로 단순화한 부드러운 민트 그린(Mint Green) 곡선이 중앙에 배치되고, 그 곡선 안쪽(목구멍/중앙 위치)에 딸꾹질을 막아주는 피치 오렌지(Peach Orange) 컬러의 방패가 떠 있는 형상.
    *   **디자인 특징:** 구체적인 신체 부위 묘사를 제거하여 거부감을 없애고, 모든 선과 면의 끝을 둥글게 처리하여 귀여운 느낌 강조.
*   **스플래시 화면 (Splash Screen):**
    *   앱 실행 시 "The Gentle Shield" 로고가 중앙에서 부드럽게 나타나며 메인 화면으로 자연스럽게 전환되는 애니메이션 구현.
*   **전체 UI 톤앤매너:** 메인 화면의 포인트 컬러와 로고의 파스텔 톤이 조화를 이루도록 전체 UI의 채도를 미세 조정. (Flutter/크로스 플랫폼 개발 시 동일하게 적용)

## 2. 메인 화면 구성 (Single-View UI)
앱 실행 시 첫 화면에 3개의 대형 카드 버튼을 배치하여 인지 부하를 최소화함. (Deep Navy / Charcoal Black 배경 베이스)

### [버튼 1] 숨참기 (Vacuum Hold)
*   **상황 문구:**
    *   (KR) "물이 없을 때 · 어디서나 조용히<br>클래식한 방법이죠"
    *   (EN) "No water? Stay quiet. The classic way."
*   **포인트 컬러:** 차분한 Sky Blue
*   **과학적 원리:** 이산화탄소($CO_2$) 농도를 높여 횡격막 경련 리듬을 강제로 중단함.
*   **구현 기능:**
    *   **5-15-5 타이머:** 흡입(5초) -> 정지(15초) -> 배출(5초) 가이드 애니메이션.
    *   **햅틱 가이드:** 숨을 참는 구간에서 미세한 진동을 주어 사용자가 타이머를 보지 않아도 박자를 맞출 수 있게 함.

### [버튼 2] 90도 저중력 음수 (Silent Sip)
*   **상황 문구:**
    *   (KR) "물은 있고 창피한 건 싫을 때<br>사실 그냥 물 마신다는 의미예요"
    *   (EN) "Water's here, but keep it low-key. Just a sip."
*   **포인트 컬러:** 안정적인 Mint Green
*   **과학적 원리:** 역방향 자세로 물을 삼켜 미주신경(Vagus Nerve)을 강하게 자극.
*   **구현 기능:**
    *   **기울기 센서 연동:** 스마트폰 가속도 센서를 활용하여 사용자가 상체를 90도 가깝게 숙였을 때만 ‘마시기 활성화’ UI 표출.
    *   **동작 가이드:** "허리를 숙여 컵의 반대편 물을 조금씩 마시세요"라는 텍스트 및 일러스트 가이드.

### [버튼 3] 동서남북 (Master Ritual)
*   **상황 문구:**
    *   (KR) "현재까지 성공률 99.9% · 소리 내도 괜찮을 때<br>제작자 추천 필살기"
    *   (EN) "99.9% Success Rate. Loud and Proud. Creator's Choice."
*   **포인트 컬러:** 에너지 있고 확실한 느낌의 Vibrant Orange
*   **과학적 원리:** 다방향 회전, 음성 출력(성대 자극), 인지적 주의 분산을 통한 신경계 리셋.
*   **구현 기능:**
    *   **가상 나침반 (Virtual Compass Logic):** 실제 북쪽이 어디든 상관없이, 사용자가 버튼을 누른 시점의 정면을 'East(동쪽)'으로 강제 고정.
        *   "자, 이제부터 당신이 보는 쪽이 동쪽입니다(Now, this is East)"라는 안내 문구와 함께 상대적인 회전값(90도씩)만 감지.
    *   **음성 인식 인터랙션:** 사용자가 특정 방향을 향하고 "동!", "서!" 등을 외치면 마이크로 소리를 감지하여 다음 단계로 자동 전환.
    *   **게이미피케이션:** 4방향을 모두 완료하면 성공 대시보드로 이동.

## 3. 디자인 및 기술적 요구 사항 (Design & Technical Specs)

### Typography & Visual Tone
*   **폰트 (Fonts):**
    *   **국문:** 가독성이 높고 현대적인 'Pretendard' 또는 'Suit' (살짝 곡률이 있는 산세리프).
    *   **영문:** 글로벌 감성의 'Inter' 또는 'Montserrat'. 국문과 이질감이 없도록 두께 조절.
    *   **강조:** 버튼 내 핵심 키워드는 Bold 혹은 Extra Bold를 사용하여 패닉 상황에서의 가독성 극대화.
*   **컬러 (Colors):**
    *   **배경:** 심리적 안정감을 주는 Deep Navy 또는 Charcoal Black.
    *   **포인트:** Sky Blue (숨참기), Mint Green (90도 음수), Vibrant Orange (동서남북).

### UX/UI & 센서 제어
*   **Micro-interaction:** 버튼 터치 시 내가 '선택했다'는 확신을 주는 부드러운 피드백 애니메이션.
*   **Zero-Inertia:** 앱 실행 후 기능 진입까지 1초 이내 지연 시간 달성.
*   **센서 활용:** Accelerometer(90도 각도 체크), Gyroscope(상대적 방향 트래킹), Audio Engine(음성 데시벨 감지).
*   **Visual Feedback:** 각 단계 성공 시 시각적 확인(Check) 효과 제공.

## 4. 상세 페이지 기능 정의 (Sub-Page Detailed Flow)
모든 후속 페이지는 `[나가기(X)]` 버튼을 좌측 상단에 배치하여 언제든 메인으로 돌아올 수 있게 설계합니다.

### ① [Page: Vacuum Hold] 숨참기 인터페이스
*   **UI 구성:** 중앙에 거대한 '브리딩 서클(Breathing Circle)' 배치.
*   **인터랙션 텍스트 가이드:** (타이머와 연동되어 호흡을 리드)
    *   **준비:** "등을 펴고 편안하게 앉으세요."
    *   **Inhale (5s):** 서클이 커지며 "5초간 깊게 숨을 들이마십니다... (5, 4, 3...)"
    *   **Hold (15s):** 서클 멈춤. "멈추세요! 15초간 유지합니다. (진동에 집중하세요)" (1초마다 짧은 진동)
    *   **Exhale (5s):** 서클이 작아지며 "천천히, 끝까지 내뱉으세요... (5, 4, 3...)"
    *   **종료:** "몸의 긴장을 풀고 잠시 기다려보세요."
    *   **완료 시:** `[5. 결과 피드백]` 화면으로 자동 전환.

### ② [Page: Silent Sip] 90도 음수 인터페이스
*   **UI 구성:** 화면 상단에 90도 자세를 한 캐릭터 아이콘, 하단에 스마트폰 기울기를 보여주는 '수평계(Leveler)' 배치.
*   **인터랙션 텍스트 가이드:** (기울기 센서 연동)
    *   **자세 잡기 (Posture Check):** "컵을 들고 허리를 90도로 숙여주세요."
    *   **각도 체크:** (수평계 이동 중) "조금 더... 조금 더 숙여보세요."
    *   **마시기 (Activation & Action):** (각도 도달 시 화면 Mint Green 반전) "지금입니다! 컵 반대편의 물을 조금씩 마셔보세요."
    *   **정렬 (종료):** 완료 후 "천천히 상체를 세우고 물을 삼키세요."

### ③ [Page: Master Ritual] 동서남북 챌린지 인터페이스
*   **UI 구성:** 화면 전체를 4등분한 레이아웃. 현재 방향을 강조하는 나침반 휠 배치.
*   **인터랙션 텍스트 가이드 (동→서→남→북):**
    *   **진입 시 (Orientation):** "준비되셨나요? {User_Name}님의 정면을 '동쪽'으로 설정합니다." -> `[확인, 준비 완료!]` 버튼 터치
    *   **Mission 1 [East]:** "정면을 보고 크게 외치세요: '동!'" / (인식 성공 시) "좋아요! 이제 180도 뒤로 돌아보세요."
    *   **Mission 2 [West]:** "반대편을 보고 외치세요: '서!'" / (인식 성공 시) "완벽합니다. 이제 왼쪽으로 90도만 돌까요?"
    *   **Mission 3 [South]:** "왼쪽을 보고 외치세요: '남!'" / (인식 성공 시) "거의 다 왔어요! 다시 180도 뒤를 봅니다."
    *   **Mission 4 [North]:** "마지막입니다! 크게 외치세요: '북!'" / (인식 성공 시) "미션 클리어! 이제 정면을 보고 숨을 고르세요."
    *   **진행 바:** 상단에 4단계 게이지(E-W-S-N) 배치.
*   **수동 패스 기능 (Fallback):** 공공장소 등 시끄러운 환경에서 음성 인식이 어려울 경우를 대비해, 화면 중앙의 마이크 아이콘을 길게 누르거나 특정 영역을 터치하면 다음 단계로 즉시 넘어갈 수 있는 '수동 패스' 백업 기능을 제공합니다.

## 5. 공통 결과 및 피드백 (Result & Feedback Screen)
모든 가이드가 종료된 후 데이터 축적을 위한 최종 페이지입니다.
*   **질문:** "딸꾹질이 멈췄나요? (Did it stop?)"
*   **선택지:** 
    *   `[YES! 멈췄어요]`: "축하합니다! 이번 성공을 기록했습니다." (홈으로 이동)
    *   `[아니요, 아직요]`: "포기하지 마세요! 다른 방법을 시도해볼까요?" (메인 화면으로 이동하여 다른 모드 추천)
*   **데이터 기록:** 선택한 모드와 성공 여부를 로컬 DB에 저장하여 개인화 알고리즘에 반영.

### 5-1. 보너스: 상황별 위트 텍스트 (Micro-copy)
개인화 변수(`{User_Name}`, `{Badge_Name}`)를 활용하여 유대감을 형성하는 알림 및 팝업 텍스트입니다.
*   **온보딩 완료 시:** "반가워요, {User_Name}님! 딸꾹질에서 해방되시죠."
*   **딸꾹질 멈춤 확인:** "{User_Name}님, 이제 조용해졌나요? (Did it stop?)"
*   **실패 후 재시도 유도:** "이런, 끈질긴 녀석이네요. 다른 무기(모드)를 써볼까요?"
*   **뱃지 획득 시:** "축하합니다! [{Badge_Name}] 뱃지가 잠금 해제되었습니다!"

## 6. 마이 리포트: 변수 연동 (Dynamic Data in My Report)
이제 스와이프 업을 했을 때 나타나는 리포트의 모든 문구는 고정된 이름이 아닌, 각 사용자가 설정한 이름으로 치환되어 보여집니다.

### 6-1. 헤더 영역 (Header Section)
*   **출력 예시:**
    *   (사용자가 '민수'라고 입력 시): "민수님, 현재까지의 방어 성공률은 94%입니다."
    *   (사용자가 'Sarah'라고 입력 시): "Sarah, your current shield success rate is 94%."
*   **Insight Message:** "현재 {User_Name}님에게는 [동서남북] 방법이 가장 효과적이에요!"

### 6-3. 중앙: 성공률 분석 차트 (Success Analytics)
*   **Visual:** 3가지 모드의 포인트 컬러를 활용한 도넛 차트(Donut Chart).
*   **Data Legend:**
    *   Sky Blue: 숨참기 (Vacuum Hold)
    *   Mint Green: 90도 음수 (Silent Sip)
    *   Vibrant Orange: 동서남북 (Master Ritual)
*   **Interactivity:** 차트의 각 섹션을 터치하면 해당 방법으로 몇 번 성공했는지 구체적인 횟수가 중앙에 표시됩니다.

### 6-4. 하단: 히스토리 로그 (History Timeline)
최근 딸꾹질 발생 기록을 타임라인 형태로 보여줍니다.
*   **Layout:** 수직 타임라인.
*   **Data Point:**
    *   Left: 날짜 및 시각 (Ex: 05.10 23:15)
    *   Center: 사용 모드 아이콘 (🧭, 🥤, 🫁)
    *   Right: 결과 배지 (Success ✅ / Fail ❌)
*   **Action:** 기록을 길게 누르면 삭제하거나 메모(예: '매운 음식 먹은 후 발생')를 추가할 수 있는 옵션 제공.

### 6-5. 수집 및 정복 배지 (Achievement Badges)
사용자의 지속적인 앱 사용을 유도하는 게임화(Gamification) 요소입니다.
*   **Hiccup Killer (딸꾹질 킬러):** 딸꾹질 10회 연속 방어 성공 시 활성화. (아이콘: 방패와 칼)
*   **Master of East-West (방향의 지배자):** 동서남북 루틴 5회 완수 시 부여. (아이콘: 왕관을 쓴 나침반)
*   **Silent Ninja (침묵의 닌자):** 90도 음수 방법으로 공공장소에서 3회 이상 성공 시 부여. (아이콘: 닌자 마스크)
*   **First Shield (첫 방어):** 앱 가입 후 첫 성공 시 즉시 부여.

## 7. 뱃지 시스템: 개인화 및 확장 (Personalized Badges)
뱃지를 확인할 수 있는 전용 페이지와 추가 뱃지 종류를 안티그래비티 팀이 바로 구현할 수 있게 정리했습니다.

### 7-1. [Page] 뱃지 아카이브 (The Collection)
*   **진입:** 마이 리포트 내 `[전체 뱃지 보기]` 버튼.
*   **구성:** 획득한 뱃지는 컬러로, 미획득 뱃지는 잠금(회색조) 처리된 3xN 그리드 레이아웃.

### 7-2. 추가 뱃지 리스트 (Badge Expansion)
개발팀이 뱃지 이미지를 제작할 수 있도록 컨셉을 세분화했습니다.

*   **첫 번째 방패 (The First Shield):** 앱 설치 후 첫 성공 (미니멀한 방패)
*   **동쪽의 지배자 (East Master):** 동서남북 루틴 5회 성공 (나침반과 왕관)
*   **심야의 수호자 (Night Owl):** 밤 12시 이후 성공 (초승달과 방패)
*   **침묵의 마스터 (Silent Ninja):** 90도 음수 10회 성공 (닌자 복면 또는 물방울)
*   **폐활량 왕 (King of Lungs):** 숨참기 모드 15회 성공 (폐 아이콘)
*   **전설의 가드 (Legendary Guard):** 모든 방법으로 각 10회 이상 성공 (황금빛 아우라 뱃지)

### 7-3. 뱃지 활성화 로직 (Badge Activation Logic)
이 부분은 개발팀이 백엔드/로컬 DB에서 어떻게 조건을 판별할지 정의하는 핵심 명세입니다.

**1. 실시간 이벤트 리스너 (Event Listener)**
모든 가이드 종료 후 사용자가 `[YES! 멈췄어요]` 버튼을 누르는 순간, 시스템은 다음의 데이터를 즉시 대조합니다.
*   `Total_Success_Count` (누적 성공 횟수)
*   `Current_Mode_Success_Count` (현재 모드 성공 횟수)
*   `Timestamp` (발생 시각)
*   `Success_Streak` (연속 성공 횟수)

**2. 조건 충족 시 인터랙션 (Trigger & Animation)**
*   **즉각적 보상:** 조건이 충족되는 즉시 결과 화면 위에 "새로운 뱃지 획득!" 팝업을 띄웁니다.
*   **시각적 변화:** 
    *   뱃지 아카이브 페이지에서 흑백(Grayscale) 처리되었던 아이콘이 고유의 포인트 컬러로 변하며 활성화됩니다.
    *   활성화된 뱃지는 'New' 배지가 붙으며 진열장 가장 앞줄로 정렬될 수 있습니다.

**3. 활성화 상세 명세 (Activation Criteria Examples)**
개발팀이 코딩할 때 참고할 수 있도록 조건을 수치화했습니다.
*   **첫 번째 방패:** `Total_Success_Count == 1`
*   **동쪽의 지배자:** `Mode_Master_Ritual_Success >= 5`
*   **심야의 수호자:** `Success_Time`이 `00:00~06:00` 사이일 때
*   **침묵의 닌자:** `Mode_Silent_Sip_Success >= 10`
*   **불굴의 의지:** 한 세션 내에서 Fail 후 다시 시도하여 Success한 경우

## 8. 안티그래비티 개발팀을 위한 최종 가이드

**[변수 및 데이터 관리 핵심]**
*   **String Variable:** 모든 텍스트 UI는 하드코딩하지 말고, 온보딩에서 입력받은 String `{User_Name}` 변수를 참조하도록 설계해 주세요.
*   **Naming Fallback:** 만약 사용자가 이름을 입력하지 않거나(공백) 오류가 날 경우를 대비해 기본값(Default)을 "사용자" 또는 **"User"**로 설정해 주세요.
*   **Badge Page:** 뱃지 아카이브는 스크롤이 가능한 페이지로 구성하고, 뱃지 터치 시 획득 조건과 `{User_Name}`님이 획득한 날짜가 팝업으로 노출되게 해주세요.

**[뱃지 활성화 구현 가이드]**
*   **State Management:** 각 뱃지의 활성화 상태(`is_locked`: true/false)를 로컬 DB에 저장하여 앱을 껐다 켜도 유지되게 해주세요.
*   **Confetti Effect:** 뱃지 활성화 팝업이 뜰 때, 화면에 화려한 축하 효과(Confetti)를 주어 사용자가 **'딸꾹질을 멈춘 보상'**을 확실히 인지하게 해주세요.
*   **Haptic Feedback:** 뱃지 획득 순간에 기분 좋은 '두 번의 짧은 진동'을 주어 촉각적 쾌감을 제공해 주세요.

## 9. 설정 및 데이터 관리 (Settings & Data Management)
앱의 메인 화면 우측 상단 또는 마이 리포트 내의 '설정(⚙️)' 탭을 통해 접근할 수 있는 관리 메뉴입니다.
*   **프로필 수정:** 온보딩 시 입력했던 이름(`{User_Name}`)을 언제든 다시 변경할 수 있는 기능.
*   **데이터 리셋 (Data Reset):** "모든 기록 지우기" 버튼. 클릭 시 누적 성공 횟수, 연속 성공 기록, 뱃지 활성화 상태 등 로컬 DB에 저장된 사용자의 모든 데이터를 초기화하는 기능. (테스트 목적 및 사용자 프라이버시 보호)
