# [최종 보고서] 프로젝트: 딸꾹뚝 (Hiccup Stop)

## 1. 프로젝트 개요
*   **과제명:** 상황별 최적의 딸꾹질 완화 알고리즘을 제공하는 ‘인터랙티브 케어 가이드’ 개발
*   **개발 목적:** 딸꾹질 발생 시 당황하는 사용자에게 과학적 원리에 기반한 3가지 해결책을 직관적인 UI와 센서 인터랙션을 통해 제공하여 즉각적인 완화를 돕는 것
*   **핵심 가치:** 전문성(Scientific), 직관성(Intuitive), 재미(Gamification)

## 2. 기술 스택 (Tech Stack)
*   **Framework:** Next.js (App Router)
*   **Language:** TypeScript
*   **Styling:** Tailwind CSS (v4), Vanilla CSS (Custom Animations)
*   **Web APIs:** 
    *   `DeviceOrientationEvent`: 스마트폰 기울기 감지 (90도 음수 모드)
    *   `Web Speech API`: 음성 인식 (동서남북 모드)
    *   `Vibration API`: 햅틱 피드백 제공 (타이머 및 보상)
*   **Storage:** LocalStorage (사용자 데이터 및 히스토리 관리)

## 3. 주요 기능 및 구현 상세

### 3-1. 온보딩 및 개인화 (Onboarding)
*   사용자 이름을 입력받아 앱 내 모든 UI 문구에 `{User_Name}` 변수를 동적으로 반영.
*   로컬 스토리지를 활용하여 재방문 시 스플래시 화면 후 즉시 메인으로 진입하는 UX 구현.

### 3-2. 인터랙티브 완화 모드 (3-Core Methods)
1.  **숨참기 (Vacuum Hold):**
    *   과학적 원리: 이산화탄소($CO_2$) 농도를 높여 횡격막 경련을 중단.
    *   구현: 5-15-5 브리딩 타이머와 확장/축소 애니메이션 서클을 통해 호흡 리드.
    *   **iOS 대응:** Vibration API 미지원 환경(iOS Safari)을 고려하여 타이머 매 초마다 시각적 펄스(Visual Pulse) 애니메이션을 폴백(Fallback)으로 제공하여 사용자 피드백 강화.
2.  **90도 음수 (Silent Sip):**
    *   과학적 원리: 역방향 자세로 미주신경(Vagus Nerve) 자극.
    *   구현: 스마트폰 가속도/자이로 센서를 활용하여 사용자가 75~90도 이상 숙였을 때만 가이드가 활성화되는 ‘상태 감지’ 로직 적용.
    *   **권한 및 오디오 관리:** iOS 및 최신 브라우저의 보안 정책을 고려하여, 모드 진입 전 [준비 완료] 버튼 클릭 시 센서 권한(`requestPermission`)과 오디오 컨텍스트 활성화를 동시에 처리하는 매끄러운 UX 설계.
3.  **동서남북 (Master Ritual):**
    *   과학적 원리: 다방향 회전 및 음성 출력을 통한 신경계 리셋.
    *   구현: 가상 나침반 UI와 실시간 음성 인식을 결합. 사용자가 특정 방향을 보고 구호를 외치면 센서와 마이크가 이를 감지하여 다음 단계로 자동 전환.
    *   **상대적 방위(Relative Heading):** 실제 자북(Magnetic North)과 관계없이, 사용자가 모드를 시작하는 시점의 방향을 상대적 0도(East)로 강제 고정하여 실내외 어디서든 정확한 90도 회전을 감지할 수 있도록 설계.
    *   **수동 패스(Manual Fallback):** 시끄러운 환경이나 음성 인식이 어려운 상황을 대비해, 화면 터치만으로 다음 단계로 진행할 수 있는 '수동 패스' 백업 기능을 제공하여 UX 끊김 방지.
    *   **글로벌 인식 로직:** Web Speech API를 통해 한국어(`ko-KR`)와 영어(`en-US`)를 모두 지원. "동!"과 "East!"와 같은 각 언어별 핵심 키워드를 인식.

### 3-3. 데이터 분석 및 리포트 (My Report)
*   **피드백 루프 및 데이터 수집:** 모든 완화 모드 종료 후 "딸꾹질이 멈췄나요? (Yes/No)" 질문을 통해 사용자 피드백을 직접 수집. 해당 응답은 `LocalStorage`에 기록되어 리포트의 성공률 분석에 즉각 반영됨.
*   **성공률 분석:** 수집된 피드백 데이터를 기반으로 각 모드별 성공 비중을 도넛 차트로 시각화.
*   **개인화 인사이트:** 사용자의 과거 데이터를 대조하여 가장 성공률이 높은 최적의 모드를 추천.
*   **히스토리 로그:** 타임라인 UI를 통해 발생 시간, 사용 모드, 성공 여부를 파악.
*   **Privacy by Design:** 모든 사용자 데이터(이름, 발생 기록)는 외부 서버로 전송되지 않고 사용자의 기기(`LocalStorage`)에만 안전하게 저장됨을 명시하여 데이터 주권 및 프라이버시 보호 강화.

## 4. 기술적 최적화 및 인프라 사양

### 4-1. 보안 컨텍스트 및 배포 환경
*   **HTTPS 필수 적용:** Web Speech API 및 센서 API는 보안 환경에서만 활성화되므로, 서비스 배포 시 SSL 인증서가 적용된 보안 컨텍스트(HTTPS) 구축을 필수 권장 사항으로 정의.

### 4-2. 디자인 시스템 성능 최적화
*   **글래스모피즘 가속:** `backdrop-blur` 효과로 인한 성능 저하를 방지하기 위해 GPU 가속(`transform: translateZ(0)`)을 활용하고, 저사양 기기에서도 부드러운 애니메이션이 유지되도록 블러 강도를 최적화(12px)함.

### 4-3. 사용자 접근성 및 PWA 도입
*   **설치형 웹 경험 (PWA):** Next.js 환경에서 PWA를 구현하여 사용자가 홈 화면에 아이콘을 추가하고 앱처럼 즉각 실행할 수 있게 함. 
*   **오프라인 가용성:** 로컬 스토리지를 활용한 데이터 관리 덕분에 네트워크 연결이 불안정한 환경이나 오프라인 상태에서도 핵심 완화 가이드와 리포트 기능을 차질 없이 사용할 수 있는 높은 안정성 확보.

## 5. 결론 및 기대 효과
본 프로젝트는 스마트폰의 센서 및 Web API를 활용하여 사용자의 실질적인 신체 액션을 유도하는 차세대 인터랙티브 헬스케어 모델을 제시합니다. iOS의 기술적 제약과 보안 환경을 고려한 설계(Fallback, Permission Guide)를 통해 실질적인 사용성을 확보하였으며, '딸꾹뚝' 프로젝트를 통해 데이터 기반의 정밀 솔루션으로의 확장 가능성을 입증하였습니다.
